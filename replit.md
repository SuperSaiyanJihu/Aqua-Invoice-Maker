# Excel Aquatics Invoice Management

## Overview

This is an invoice management system for Excel Aquatics, a swimming lessons business in Colonie, NY. The application allows managing students, creating invoices with attendance tracking, generating PDF invoices, and viewing invoice history. It's a full-stack TypeScript application with a React frontend and Express backend, backed by PostgreSQL.

## User Preferences

Preferred communication style: Simple, everyday language.

## System Architecture

### Frontend
- **Framework**: React with TypeScript, bundled by Vite
- **Routing**: Wouter (lightweight client-side router)
- **State/Data Fetching**: TanStack React Query for server state management
- **UI Components**: shadcn/ui (new-york style) built on Radix UI primitives with Tailwind CSS
- **Styling**: Tailwind CSS with CSS variables for theming (light/dark mode support)
- **Structure**: Single-page app with three tabbed views — Create Invoice, Students, Invoice History
- **Path aliases**: `@/` maps to `client/src/`, `@shared/` maps to `shared/`

### Backend
- **Framework**: Express.js running on Node with TypeScript (via tsx)
- **API Pattern**: RESTful JSON API under `/api/` prefix
- **PDF Generation**: PDFKit for creating invoice PDFs server-side
- **Dev Server**: Vite dev server integrated as Express middleware (HMR via `server/vite.ts`)
- **Production**: Static files served from `dist/public`, server bundle built with esbuild to `dist/index.cjs`

### API Routes
- `GET/POST /api/students` — List or create students
- `PATCH/DELETE /api/students/:id` — Update or delete a student
- `GET/POST /api/invoices` — List or create invoices
- `DELETE /api/invoices/:id` — Delete an invoice
- `POST /api/invoices/generate` — Generate and download a PDF invoice

### Database
- **Database**: PostgreSQL (required via `DATABASE_URL` environment variable)
- **ORM**: Drizzle ORM with `drizzle-zod` for schema validation
- **Schema location**: `shared/schema.ts` (shared between client and server)
- **Migrations**: Generated by drizzle-kit to `./migrations` directory
- **Push command**: `npm run db:push` applies schema directly
- **Tables**:
  - `students` — id, full_name, class_day_time, rate_per_class
  - `invoices` — id, student_id, student_name, class_day_time, rate_per_class, attendance_dates (text array), comments, created_at
- **Seeding**: `server/seed.ts` auto-seeds 4 sample students if the table is empty

### Storage Layer
- `server/storage.ts` defines an `IStorage` interface with a `DatabaseStorage` implementation
- All database access goes through this storage abstraction

### Build System
- **Dev**: `tsx server/index.ts` runs the server with Vite middleware for HMR
- **Build**: Custom `script/build.ts` runs Vite build for client, then esbuild for server
- **Server bundling**: Key dependencies are bundled (allowlisted) to reduce cold start times; others are external

## External Dependencies

### Database
- **PostgreSQL** — Primary data store, connected via `pg` Pool with `DATABASE_URL` env var

### Key NPM Packages
- **drizzle-orm** + **drizzle-kit** — ORM and migration tooling for PostgreSQL
- **pdfkit** — Server-side PDF generation for invoices
- **express** — HTTP server framework
- **zod** + **drizzle-zod** — Schema validation (shared between client and server)
- **@tanstack/react-query** — Client-side data fetching and caching
- **react-day-picker** — Calendar component for selecting attendance dates
- **date-fns** — Date formatting and manipulation
- **wouter** — Lightweight client-side routing

### Replit-specific
- `@replit/vite-plugin-runtime-error-modal` — Runtime error overlay in development
- `@replit/vite-plugin-cartographer` — Dev tooling (development only)
